🚀 快速入门

                
① 先到「连接」页，选 TCode（串口）或 Intiface（Buttplug）方式，填好参数后点击连接。

                
② 到「设备」页确认设备已出现，用手动测试滑块验证轴向响应。

                
③ 在「配置」页检查 OSC 接收端口（默认 9001），然后在 VRChat 中启用 OSC。

                
④ 回到「总览」或顶部按钮条，点击「启动 Loop」，设备即开始实时输出。

              

              

                
📋 页面功能速查

                
总览 — 整体运行状态、服务健康检查和姿态预览。

                
连接 — TCode 串口和 Intiface Buttplug 的连接参数与操作。

                
配置 — 服务地址、安全限制、BPM 检测和信号矩阵。

                
设备 — 已连接设备的本地备注、快捷操作和手动轴测试。

                
脚本 — 录制、导出和预览 .funscript 文件。

                
监控 — 实时 OSC 参数流和运行日志。

              

              

                
⚠️ 顶部快捷操作

                
页面顶部的按钮条（启动 Loop / 停止 Loop / 紧急停止 / 清除紧急停止）在任何标签页都可用。

                
发生意外时，直接点击「紧急停止」可立即将所有输出强制归零。

                
「清除紧急停止」会恢复正常输出——请确认安全后再操作。

              

            

          


          

            
🔌 TCode 串口连接说明

            

              

                
串口号 (COM Port)

                
在 Windows 设备管理器 → 端口里找到设备对应的 COMX 号，填入此处。如果列表中没有出现，检查 USB 驱动（CH340 / CP2102 等）是否已安装，且串口没有被其他程序占用。

              

              

                
MinPos / MaxPos

                
TCode 轴位范围 0–999。MinPos 是行程下端（默认 100），MaxPos 是行程上端（默认 900）。过小的范围会限制行程；超出设备物理限制的值可能触发限位保护。

              

              

                
MaxVelocity

                
每帧最大移动量（速度上限）。数值越大动作越快，但超过设备机械限制会导致失步或异响。建议先从保守值（如 1000）开始测试，逐步调高。

              

              

                
UpdatesPerSecond

                
每秒向设备发送命令的频率。建议 50–100。过高可能导致串口缓冲区溢出；过低会使运动出现明显的分段感。

              

              

                
优先速度模式

                
勾选后发送速度指令 (Ixxxx) 代替绝对位置指令 (Lxxxx)。需要固件支持速度插值。OSR2/SR6 的常见固件通常都支持，速度模式下动作更流畅。

              

              

                
L0 反向

                
将 L0 轴输出取反（0 输出变为最大位，1 输出变为最小位）。当设备倒置安装或轴向与预期相反时使用。

              

            

          


          

            
🦾 Intiface 连接说明

            

              

                
什么是 Intiface

                
Intiface Central 是 Buttplug.io 的设备管理应用，支持通过 Bluetooth、USB 等方式驱动振动器和线性设备。Sensa 通过 WebSocket 与 Intiface 通信，不需要直接操作 Buttplug 协议。

              

              

                
WebSocket 地址

                
Intiface Central 启动后在本地监听 WebSocket。默认地址 ws://localhost:12345。如果你在另一台机器上运行 Intiface，将 localhost 换成对应 IP。

              

              

                
托管 intiface-engine

                
如果本机没有安装 Intiface Central，可在 Sensa 旁边放一个 intiface-engine.exe 文件，勾选「托管」后 Sensa 会自动启动它。适合不想打开额外图形界面的场景。

              

              

                
设备不出现怎么办

                
① 先点「连接 Intiface」建立 WebSocket 连接，再点「开始扫描」让 Intiface 搜索蓝牙/USB 设备。② 确认 Intiface Central 中设备已配对。③ 部分设备需要通电并处于配对模式才会被发现。

              

            

          


          

            
🛡️ 安全设置说明

            

              

                
GlobalIntensityCap（全局强度上限）

                
范围 0.0–1.0。所有设备的输出都不会超过这个值。设为 0.8 表示最大 80% 强度。首次使用建议设低一点（如 0.6），习惯后再调高。

              

              

                
RampUpMs（启动渐增时间）

                
Loop 启动后输出从 0 渐增到目标值所需的毫秒数。防止骤然全速启动带来的机械冲击。建议 500–2000；设为 0 则瞬时启动。

              

              

                
Idle（空闲行为）

                
Loop 停止或长时间收不到新的 OSC 参数时，设备的默认行为。通常选「回中」（Park），避免设备长时间停在非中心位置。

              

              

                
EmergencyStopKey（紧急停止热键）

                
填入一个键盘键名（如 Escape 或 F12），按下后立即触发紧急停止，将所有输出强制归零。留空则禁用热键功能。配合顶部「紧急停止」按钮使用更安全。

              

            

          


          

            
🎵 BPM 与节奏检测

            

              

                
工作原理

                
开启后系统会分析 OSC 参数的变化频率，自动估算当前动作的节拍 BPM，并在「服务状态」面板中显示。BPM 可以被某些信号映射曲线用于节奏同步输出。

              

              

                
WindowMs（计算窗口）

                
参与 BPM 计算的历史时间窗口（毫秒）。窗口越大计算越稳定，但对突然变速的响应会有延迟。推荐值 2000–5000。

              

              

                
MinBpm / MaxBpm

                
超出此范围的节奏会被视为噪声忽略掉。典型设置：MinBpm 20，MaxBpm 180，覆盖常见的动作节奏范围。

              

            

          


          

            
📡 信号矩阵说明

            

              

                
OSC Path

                
VRChat 发出的 OSC 参数路径，如 /avatar/parameters/Sensa_L0。必须与 VRChat 头像中配置的参数名完全一致。

              

              

                
Role（角色）

                
该信号控制设备的哪个维度：Depth（主冲程 L0）、Vibrate（振动）、Roll/Pitch/Twist（姿态轴）等。同一 Role 允许多个信号叠加。

              

              

                
Curve（曲线）

                
将输入值映射到输出值的方式：Linear（线性）、Ease（缓入缓出）、Step（阶梯）等。非线性曲线可以让低值时运动更轻柔、高值时更激烈。

              

              

                
Min / Max

                
VRChat 中该参数的有效输入范围（通常 0–1），超出范围的值会被裁剪后再做映射。

              

              

                
α（平滑系数）

                
EMA 指数平均平滑系数（0.01–1.0）。越接近 0 越平滑（延迟越高）；越接近 1 越贴近原始值（响应越快）。推荐 0.5–0.8。

              

              

                
Dead Zone（死区）

                
当信号变化幅度小于此值时不触发更新，避免微小抖动引起不必要的运动。设为 0.01 可过滤掉大部分噪声。

              

            

          


          

            
🔧 设备页 & 手动测试

            

              

                
设备工作台

                
展示所有已连接的真实设备（以及可选的 Mock 预览卡）。每张卡片上可以保存设备备注，记录自定义参数配置，方便以后一键还原。

              

              

                
手动测试台

                
无需 VRChat 信号，直接用滑块控制每个轴的位置： L0=主冲程（0=底端, 1=顶端），R0=横滚，R1=俯仰，R2=扭转，L1=横向，L2=偏摆，Vibrate=振动强度。

                
勾选「启用手动覆盖」→ 调整滑块 → 点击「应用手动测试」。完成后必须点「清除手动测试」才能恢复 OSC 驱动模式。

              

              

                
TCode 回中（Park）

                
将设备所有轴回到安全中点位置（通常 L0=0，R0/R1/R2=0.5）。连接后、测试前或长时间不用时建议先执行回中。

              

            

          


          

            
📝 脚本页说明

            

              

                
录制

                
点「开始录制」后，系统会将 L0 轴的实时输出持续记录为时间戳帧。适合在手动玩耍或观看视频时同步录制轨迹。

              

              

                
导出 .funscript

                
停止录制后点「导出」，系统自动做 RDP 简化，去掉冗余中间点后生成 .funscript 文件并触发下载。该格式兼容 Handy、MultiFunPlayer 等播放软件。

              

              

                
导入与预览

                
可以导入已有 .funscript 文件，在浏览器内的波形图上预览时间轴和动作节奏。预览播放不会向设备发送任何命令，仅用于可视化检查。

              

            

          


          

            
🔍 监控页说明

            

              

                
参数流

                
实时显示从 VRChat 收到的每个 OSC 参数路径、当前值、类型和时间戳。可用搜索框过滤，适合检查头像参数是否正确发出。如果表格为空，说明当前没有 OSC 数据到达。

              

              

                
日志

                
记录连接状态变化、配置保存、录制事件和运行错误等。查看日志可以帮助排查连接失败、串口问题和 Intiface 引擎异常。

              

            

          


          

            
❓ 常见问题排查

            

              

                
设备没有反应

                

                  ① 检查 Loop 是否已启动（顶部状态栏）。② 检查 Emergency Stop 是否已触发（状态栏 Emergency 变红时需点「清除」）。③ 检查 GlobalIntensityCap 是否为 0。④ 检查信号矩阵是否有参数路径匹配。
                

              

              

                
TCode 连接失败

                
① 确认 COM 口号正确（设备管理器查看）。② 确认没有其他程序占用该串口。③ 确认 USB-串口驱动已安装（CH340 / CP2102 等）。④ 尝试断开重连，或重启设备后再连接。

              

              

                
OSC 参数不来

                
① VRChat 设置 → OSC → 确认已启用。② VRChat 默认 OSC 输出端口是 9001，确认与 Sensa 配置的接收端口一致。③ 防火墙是否拦截了 UDP 9001。④ 切换头像后有时需要等待 OSC 重新初始化。

              

              

                
Intiface 找不到设备

                
① 先连接 Intiface 再点「开始扫描」。② 确认设备处于配对/广播模式（通常需要长按开关）。③ 蓝牙设备需要系统蓝牙已开启。④ 部分设备需要在 Intiface Central 里先手动配对。
