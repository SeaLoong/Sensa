# Sensa WebUI 重构工作报告

## 本轮目标

围绕 WebUI 的信息架构与实际排障体验，完成了以下目标：

- 将页面默认标题固定为 `Sensa WebUI`
- 将 `autoOpenBrowser` 从 WebUI 可编辑配置中移除，回归 Native / 后端职责
- 将页面改造成更清晰的六个功能分区：`总览 / 连接 / 配置 / 设备 / 监控 / 帮助`
- 强化“设备工作台”思路，让设备相关操作、记忆和测试集中呈现
- 在无实机环境下提供 Mock 设备卡片，便于检查依赖设备的布局与交互
- 对页面进行真实运行验证，而不是只做静态改样式

## 主要改动

### 1. 信息架构重组

页面职责重新划分如下：

- `总览`
  - 显示服务运行状态、关键诊断、姿态预览
  - 没有设备时给出下一步操作建议
  - 有设备时汇总当前连接状态与建议路径
- `连接`
  - 将 TCode 串口连接与 Intiface / Buttplug 连接拆分成独立卡片
  - 各自保留必要的连接参数、状态和快捷操作
- `配置`
  - 仅保留设备无关的全局配置
  - 服务配置、处理链配置、信号矩阵集中管理
  - 不再暴露 WebUI 标题与自动打开浏览器选项
- `设备`
  - 作为设备工作台，集中展示设备卡片、设备记忆、快捷操作、手动测试、录制导出
- `监控`
  - 继续保持参数流和日志的宽屏监视布局
- `帮助`
  - 只放工作流说明、布局设计说明、端点与参考链接
  - 不再混入运行时状态告警

### 2. 前端控制器恢复与重建

在本轮继续开发时，发现 `Companion/program/wwwroot/index.html` 与 `styles.css` 已经被重构，但 `wwwroot/app.js` 缺失，导致新页面结构只有静态壳，无法驱动实际功能。

因此重新创建了：

- `Companion/program/wwwroot/app.js`

该文件负责：

- 页面标题固定为 `Sensa WebUI`
- 标签页切换、主题切换、语言切换
- 读取 `/api/meta`、`/api/config`、`/api/state/*`、`/api/meta/serial-ports`
- 连接 WebSocket `/api/ws`
- 渲染总览、连接状态、配置表单、信号矩阵、设备卡片、监控面板、帮助信息
- 驱动手动测试、录制导出、设备快捷操作
- 支持 Mock 设备预览
- 支持设备本地记忆（别名 / 备注 / 配置摘要）

### 3. 设备卡片与 Mock 验证能力

新增设备卡片工作流：

- TCode / Intiface 使用不同视觉标签与摘要
- 每张设备卡片具备：
  - 设备类型/连接来源标签
  - 当前连接摘要
  - 本地记忆区（别名、备注、配置摘要）
  - 设备专属快捷操作
- 在无实机环境下，可显示两张 Mock 卡片：
  - `SR6 Mock Rig`
  - `Mock Linear Device`

这样可以在没有真实硬件时先验证：

- 卡片布局
- 字段排布
- 设备能力摘要
- 记忆配置流程
- 快捷操作按钮位置与视觉层级

### 4. 修复设备记忆草稿被刷新覆盖的问题

在实际联调时发现一个真实问题：

- 设备页会因 WebSocket 推送或定时刷新重新渲染卡片
- 如果用户正在输入设备别名/备注，但还没点击“记忆当前配置”，输入内容可能被页面重绘清空

本轮已修复：

- 在前端状态中加入 `deviceDrafts`
- 对设备记忆输入框增加草稿缓存
- 页面重渲染时优先回填草稿，再回填已保存的记忆内容
- 点击“记忆当前配置”或“重置记忆”后，再清理对应草稿

该修复已经过真实页面验证：

- 输入 Mock SR6 的别名与备注
- 等待一次自动刷新
- 字段内容仍然保留
- 点击记忆后内容仍能保持

## 已验证内容

本轮使用真实运行中的 Companion 服务进行了页面检查。

### 运行环境

- 系统：Windows
- 当前无 Intiface 实际运行环境
- Companion 服务可正常启动
- 启动日志中出现以下预期提示：
  - `intiface-engine.exe not found`
  - `Skipping connection attempt because embedded engine is unavailable`

这属于当前机器环境限制，不是本轮页面回归错误。

### 页面验证结果

已实际确认：

- 页面标题显示为 `Sensa WebUI`
- 顶部标签页为：`总览 / 连接 / 配置 / 设备 / 监控 / 帮助`
- `连接` 页中 TCode 与 Intiface 为独立卡片
- `配置` 页不再显示：
  - 标题编辑项
  - 自动打开浏览器选项
- `设备` 页包含：
  - 设备工作台
  - 手动功能测试
  - 录制与导出
- `帮助` 页只保留说明、端点与外部参考链接
- 打开 Mock 设备预览后，能看到两张模拟设备卡片
- Mock 设备卡片上的“记忆当前配置 / 重置记忆”流程可用
- 设备记忆输入内容在自动刷新后不会再丢失

## 当前已知限制

- 当前机器没有 Intiface 运行环境，因此本轮只能验证：
  - 连接失败时页面反馈是否清晰
  - Intiface 相关卡片布局与交互是否合理
  - Mock 设备路径是否可用于样式检查
- 仍需在真实 SR6 / 串口设备环境下进一步验证：
  - 回中动作的体感与时序
  - 速度模式与时间模式切换体验
  - 轴向定义与姿态预览的一致性
  - 手动测试与真实输出的匹配程度

## 建议的下一步

建议下一轮按以下顺序继续：

1. 接入真实 SR6 或同类 TCode 设备
2. 在 `连接` 页验证串口连接、断开、回中
3. 在 `设备` 页验证：
   - 设备别名与备注的实际使用方式
   - 手动测试对真实设备的响应
   - 设备卡片内容是否还需要更贴近 SR6 排障场景
4. 在 `监控` 页观察参数流、日志与动作反馈是否同步
5. 如有需要，再继续细化移动端布局与英文文案
